# NoteMaker PWA and App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship an installable, offline-capable NoteMaker PWA with a complete high-resolution app icon set.

**Architecture:** `vite-plugin-pwa` generates the manifest and Workbox service worker. One committed SVG master is rasterized by a deterministic Playwright script into every required icon size, and an automated release check validates the emitted PWA artifacts.

**Tech Stack:** Vite, vite-plugin-pwa, Workbox, Playwright, SVG, PNG, Vitest release scripts.

---

### Task 1: Define the Failing PWA Contract

- [ ] Add `scripts/pwa-check.mjs` to require the manifest, service worker, icon dimensions, and starter-audio precache entries.
- [ ] Add `npm run test:pwa` and verify it fails before PWA assets exist.

### Task 2: Generate the Brand Icon Set

- [ ] Add the 1024-unit SVG master using the approved minimal hardware mark.
- [ ] Add a deterministic Playwright rasterization script.
- [ ] Generate 1024, 512, 192, 180, and 32 pixel PNG assets.
- [ ] Inspect the master and raster output at large and favicon sizes.

### Task 3: Configure the PWA

- [ ] Install `vite-plugin-pwa` as a pinned development dependency.
- [ ] Configure manifest metadata, icon purposes, automatic updates, navigation caching, and starter-audio caching.
- [ ] Replace the embedded favicon with generated favicon, SVG, Apple touch icon, theme color, and manifest links.

### Task 4: Verify and Deploy

- [ ] Run unit tests, TypeScript linting, production build, PWA artifact checks, and Playwright smoke tests.
- [ ] Inspect desktop/mobile screenshots and service-worker registration.
- [ ] Commit and push `main`.
- [ ] Verify Vercel deploys from `main`, then run production smoke and PWA checks against the live URL.
